# Отчёт: P1-E — Круглосуточная цена золота + ослабление гарда UZS

Дата: 2026-07-23 · Инженер · Гейт-2 (диф)
Ветка: `feature/p1e-247-gold-price` (3 коммита). Спека: `.claude/specs/2026-07-24-p1e-247-gold-price-SPEC.md` (APPROVED, go 23.07).

## Сделано — по пунктам спеки

1. **Источник XAU — медиана токенов.** `keeper-xau.ts` переписан: цена = медиана
   PAXG+XAUT (пары к **USD**) через Alchemy Prices API (by-symbol, таймаут 30 с).
   Форма ответа верифицирована живым вызовом до кода (`currency: "usd"`
   строчными, `lastUpdatedAt` ISO). Медиана пары = среднее двух; один валидный →
   пост + `degraded to single source`; ноль → детектор.
2. **Chainlink — только детектор живости.** Никакой ценовой сверки и порогов.
   Токены мертвы + Chainlink свеж (≤23 ч) → refuse «WE are broken»; мертвы оба →
   refuse «infra outage». При живых токенах Chainlink на решение не влияет,
   в логе — информативная строка с возрастом.
3. **Один провайдер.** И Prices API, и детектор через один Alchemy-ключ
   (`MAINNET_RPC_URL` + `ALCHEMY_PRICES_URL` того же аккаунта). Резервный RPC
   не заводится.
4. **UZS-гард ослаблен** (`checkCbuAge`): возраст >3 суток → **warn + ретрансляция**;
   >14 суток → refuse «API сломан»; heartbeat (сутки) не тронут.
5. **Окна `maxAge*` не тронуты**, контракты не менялись — пост в тот же
   `XauUsdFeed`.
6. **Env**: `.env.example` — `ALCHEMY_PRICES_URL`, `PRICE_SYMBOLS`,
   `MAX_TOKEN_PRICE_AGE`, `CB_WARN_AGE_DAYS`, `MAX_CB_AGE_DAYS=14` (плейсхолдеры,
   ключа в репозитории нет). Рабочий `contracts/.env` укомплектован (игнорируется
   гитом, проверено `git check-ignore`).

## Не сделано / отложено

- Публичный экран «обе цифры рядом» — требование записано в P2-A, кода здесь нет.
- Fallback «market-hours» — в кармане, не реализован (не понадобился).

## Замечено, не трогаю (вне скоупа)

- Первый прогон keeper:xau упал в refuse «token sources dead» без единого WARN —
  разбор привёл к багу отбраковки near-future котировок (см. ниже, исправлен
  в этой же ветке). Промежуточные посты в цепи (PAXG-only 4036.60, первая
  медиана 4039.525) — следы отладки на тестнете, значения корректные.

## Как тестировал — команды и вывод

**Red→green (коммит A `7cd55ba`, стабы):**
```
1) should post the median (average of two) when both sources are valid
2) should post a single valid source with a degraded warning
3) should refuse as 'we are broken' when tokens are dead but chainlink is fresh
4) should refuse as 'infra outage' when tokens are dead and chainlink is stale or unreadable
5) should discard out-of-range and stale quotes instead of relaying them
6) should warn and relay a 5-day-old rate (holidays), not refuse
28 passing, 6 failing
```
После реализации (`de7e443`): вся сюита **198 passing**, tsc по keeper-файлам чист.

**Баг, пойманный ручным прогоном (red→green отдельно):** Prices API штампует
котировки «сейчас», его часы на секунды впереди последнего L1-блока → возраст
отрицательный → котировка отбраковывалась как мусор (флаки: зависело от фазы
блока; XAUT штампуется ~2 с позже PAXG — поэтому «degraded to single source»).
Тест «quote stamped slightly ahead of the block clock» — красный на старом коде,
фикс: отбраковываем только `age > maxTokenPriceAge` (`ec06a37`). После фикса:
**199 passing**.

## Отклонения от плана и почему

1. **`MAX_CB_AGE_DAYS` 7 → 14** — по директиве Капитана (п. 5 решений спеки).
2. **Два лишних XAU-поста на Sepolia** (PAXG-only, первая медиана) — следы
   отладки flaky-отбраковки; на значениях это не отразилось (оба корректные
   цены), roundId=4 это честно отражает.

## Вопросы Ревьюеру

1. `MAX_TOKEN_PRICE_AGE=3600` выбран рассуждением (котировки Prices API
   обновляются секундами, час — огромный запас). Не нашёл документированного
   SLA свежести Alchemy Prices — оставляю 3600 или сузить до 600?
2. Цена медианы расходится с Chainlink на ~8–15bps в будни (4039.5–4040.4 против
   4043.05). По решению Капитана сверки нет; фиксирую факт для P2-A (экран
   «обе цифры» будет показывать это расхождение публично).
