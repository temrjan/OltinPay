# Отчёт: P1-A — Киперы (heartbeat, UZS, резерв)

Дата: 2026-07-23 · Инженер · Гейт-2 (диф)
Ветка: `feature/p1a-keepers` (4 коммита). Спека: `.claude/specs/2026-07-23-p1a-keepers-SPEC.md` (ред. 2, APPROVED).

## Сделано — по пунктам спеки

1. **Heartbeat** — `keeper-lib.ts:decidePost`: публикация при `onchainAge >= HEARTBEAT_AGE`
   (дефолт 1800) независимо от дельты; проверка возраста строго ДО раннего выхода.
   Доказательство — красный→зелёный (см. «Как тестировал»).
2. **БЛОКЕР 1 (дыра источника)** — `MAX_SOURCE_AGE` дефолт `259200` (72 ч), градация
   `checkSourceAge`: ≤23 ч ok / (23 ч, 72 ч) warn `source heartbeat exceeded` /
   >72 ч refuse `source broken`. Пол 26 ч зафиксирован в шапке keeper-xau.
3. **БЛОКЕР 2 (nonce)** — `keeper-all.ts`: XAU → резерв → UZS последовательно одним
   процессом, per-feed try/catch, код выхода = худший. Проверено живьём (два прогона).
4. **МИНОР 3 (часы)** — `chainNowSeconds`: L2-блок для ончейн-возраста, мейннет-блок
   для возраста Chainlink; системные часы только для календарной `Date` ЦБ.
5. **МИНОР 4 (коды возврата)** — 0/2/1 во всех трёх киперах и в раннере; причины в логе.
6. **keeper-uzs.ts** — CBU JSON (`Rate`×1e8, dec 8), гард возраста 7 дней (МИНОР 1),
   heartbeat 86400, гард скачка 10%. Таймаут fetch 30 с (находка само-ревью: голый
   fetch мог повесить `keeper:all`).
7. **keeper-reserve.ts** — `RESERVE_GRAMS` (dec 0), heartbeat 1800, **БЛОКЕР 3**: гард
   10% + `RESERVE_CONFIRM` == точному значению; первый пост без гарда (сид).
8. **Замер Chainlink** — фон: семплер на 7demo (`/root/chainlink-xau-observe.sh`,
   cron `23 */6 * * *`, лог `/root/chainlink-xau-updatedAt.log`). Первая точка
   23.07 17:07 UTC: возраст записи ≈ 26 мин. Дефолт 72 ч зафиксирован из
   документированного heartbeat (22 ч 20 мин), наблюдение — подтверждение, не ворота.
9. **Тесты** — 22 новых юнит-теста (`test/keeper.test.ts`), вся сюита 186 passing.
10. **npm-скрипты** `keeper:xau/uzs/reserve/all`; `.env.example` — все новые ключи
    (имена и комментарии, без значений; адрес XAU-фида верифицирован ончейн:
    `description()` == "XAU / USD").

## Отклонения от плана и почему

1. **npm-скрипты киперов переведены с `hardhat run` на `ts-node`.** Под
   `hardhat run` `require.main === module` ложно (main — CLI hardhat), гард
   авто-запуска не сработал бы и `keeper-all` не смог бы импортировать `run()`.
   HRE киперам не нужен; `dotenv/config` подключён в самих скриптах (раньше env
   грузился побочно через hardhat.config). Запуски деплоев не тронуты.
2. **Формат ошибок киперов: throw → коды возврата** — контракт МИНОР 4 для
   алертинга P4-A (было: любой отказ = process.exit(1) без различения).
3. **Файл `.env.example` пополнен адресами канонических аттесторов** — публичные
   ончейн-адреса, не секреты; адрес XAU-фида верифицирован `description()`.
