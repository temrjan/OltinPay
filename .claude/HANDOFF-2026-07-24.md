**Правила, которые из репозитория не выводятся, а нарушать их дорого:**
- в коммитах и PR **не ставим AI-подписи** (`Co-Authored-By` и подобное) — это
  требование Капитана, оно перекрывает поведение инструментов по умолчанию;

  Капитана к Инженеру, иначе ревью перестаёт быть независимым, а незакоммиченная
  работа Инженера теряется.

## Как работаем

Роли: Капитан решает и ратифицирует, Инженер ведёт один трек за раз, Ревьюер
гейтит. Связь между Инженером и Ревьюером — только через Капитана.

Каждый PR: одна причина изменения, два гейта для кода и один для ops/доков,
доказательство работы приложено (для кода — красный тест или мутация, для ops —
чтение состояния «до/после» плюс строка отката). Мержим по зелёному CI.

Приоритет при столкновении фаз помечен **★**; второй трек встаёт на паузу, а не
идёт «параллельно» — чтобы отставание было видно сразу.
