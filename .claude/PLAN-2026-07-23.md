## 1. Критический путь — одной фразой

Пока **ни один аттестор не опубликовал значение**, `Exchange` ревертит, OLTIN не
существует, кошелёк показывает нули, gasless-смоук невозможен. Значит первым
идёт не переезд, не фронт и не заявка, а **обеспечение**. Ревьюер пришёл к тому
же выводу независимо — принимаю его формулировку: 4d-ops вперёд переезда.

```
фиды+резерв → OLTIN существует → кошелёк и панель банка показывают правду
   → видео и заявка → подача
переезд сервера и 4b идут параллельно и никого не блокируют
```

## 3. Что делаем каждым PR — правило

Один PR — одна причина изменения. Код проходит два гейта, ops и доки — один.
Каждый PR несёт **доказательство работы**: для кода красный тест или мутация;
для ops — чтение состояния «до/после» и строка отката. Мержим по зелёному CI,
адреса и решения фиксируем тем же PR.

## 4. Линия отреза — что режем первым, если не успеваем

По убыванию готовности жертвовать:

1. **P3-D** (демо-депозит Uzcard/Humo, стейкинг в UI) — показываем через панель банка.
2. **P3-C** (money-path в мини-аппе) — **fallback:** кошелёк показывает балансы из
   цепи (уже работает), а покупку/продажу демонстрируем из панели банка и
   подтверждаем в эксплорере. Демо остаётся честным, теряется «клиент нажал сам».
3. **Ступень 3 демо-банка** (автопилот сценария) — вместо неё записанное видео.
4. **R-8, R-5** (косметика и уборка) — переносятся за конкурс.

**Не режется никогда:** фаза 1 (без неё нет демо), P2-A (ссылку даём жюри),
P5-0/P5-B (без заявки конкурс не существует), откат из §0.

## 6. Риски и триггеры

| Риск | Триггер → что делаем |
|---|---|
| Фид протух в момент показа | мониторинг шлёт сигнал → ручная публикация; на странице честное «выпуск приостановлен» |
| Курс паймастера истёк 21.08 | календарное напоминание → `setRate` (одна транзакция) |
| Кончился ETH на киперах | порог остатка → долив; расход посчитан, месяц покрыт |
| 4c не успевает | линия отреза §4, п.2 — демо через панель банка |
| Переезд ломает прод | откат на образы `pre-cutover-*` из §0 |
| Требования подачи оказались тяжелее | ответ P5-0 сдвигает фриз раньше, режем §4 глубже |

## 7. Чего в плане нет намеренно

Мейннет, реальный кастодиан и лицензия, мультиассет, нативное мобильное,
ИИ-функциональность, escrow вывода, EIP-191 ownership-proof, батч-апсерт
индексера, чистка `artifacts-zk` из гита. Всё это — прод-хардненинг после
конкурса; в бэклоге записано, из плана исключено сознательно.
