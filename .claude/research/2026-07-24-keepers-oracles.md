## Q1 — staleness-лимиты: детали и правило решения для 72 ч

- **Aave v3**: `getAssetPrice` читает `latestAnswer()`, таймстемп не запрашивается
  вовсе (VERIFIED, код). **Compound v3 (Comet)**: `latestRoundData()` без чтения
  `updatedAt` (VERIFIED, код). **Morpho**: чека нет осознанно — комментарий в коде
  «assumed that the Chainlink feed keeps its promises» (VERIFIED, код).
  Де-факто допуск крупнейших потребителей = 1× heartbeat (~24 ч для XAU).
- **Synthetix v2**: глобальный `rateStalePeriod` = **90 000 с (25 ч)** — прочитано
  eth_call'ом из SystemSettings мейннета; при heartbeat 24 ч это 1.04× (VERIFIED).
  Synthetix v3 — per-node `stalenessTolerance` (механизм VERIFIED, значения не
  добыты). **GMX v2**: 300 с/30 с — но это low-latency Data Streams, не сравнение.
- Вывод агента: 72 ч = 3.2× heartbeat — по отраслевым меркам слабовато: при
  молчаливом отказе Chainlink кипер трое суток ретранслирует протухшую цену,
  маскируя аварию «свежестью» нашего фида.
- **Ключевое уточнение, снимающее контраргумент про выходные:** heartbeat
  Chainlink (22 ч 20 мин) срабатывает и при закрытом рынке — значит легитимный
  возраст записи ≤ ~23 ч всегда, и порог 26 ч **не будет ложно стрелять по
  выходным** (в отличие от выкладки «49 ч заморозки», которую первоисточник
  опроверг). Это подтвердит или опровергнет фоновое наблюдение пт→пн.
- **Правило решения (предлагаю Капитану):** если наблюдение выходных покажет
  максимум разрыва ≤ 23 ч → `MAX_SOURCE_AGE` снижаем 72 ч → **26 ч** отдельным
  PR дефолтов (прецедент Synthetix 25 ч); если разрыв > 23 ч → остаётся 72 ч с
  записанным обоснованием.

## Q2 — закрытый рынок (единственный вопрос, меняющий конструкцию)

**Развилка индустрии реальна, но «торгуем по замороженной пятничной цене» не делает
никто из найденных:**

**Лагерь «останавливаем торговлю»:**
- **Synthetix** — SCCP-83 (2021): форекс- и товарные синтетики (включая sXAU) заморожены,
  пока базовый рынок закрыт. Мотивация дословно наша: замороженный оракул + ожидаемый
  гэп на открытии = фронтраннинг переоткрытия, протокол — систематически проигрывающая
  сторона. До SCCP-83 Synthetix торговал 24/7 по замороженной цене — задокументированный
  вектор атаки. VERIFIED (sips.synthetix.io/sccp/sccp-83).
- **Ostium** (RWA-перпы, XAU): «traditional assets follow their native market hours»;
  вне часов рыночные ордера отклоняются, лимитные — в очередь до открытия. VERIFIED
  (docs.ostium.com).

**Лагерь «круглосуточный заменитель цены»:**
- **Pyth Indices** (июнь 2026): проприетарные 24/7-индексы (золото, нефть, акции) ровно
  под эту проблему; запущены dYdX, Kraken, Coinbase, Nado. Прямая цитата Kraken:
  «a perpetual needs a 24/7 reference price». VERIFIED (два источника).
- **Hyperliquid/trade.xyz**: вне сессии цена идёт со своего стакана. LIKELY (вторичные).

**Уровень оракула:** Chainlink Data Streams несёт `marketStatus` в каждом репорте —
сам вендор проектирует под «потребитель обязан остановиться по флагу»; а classic
push-фид XAU/USD официально «рыночные часы only», для выходных Chainlink предлагает
24/7 Blended Feeds. VERIFIED (docs.chain.link/data-streams/market-hours,
docs.chain.link/data-feeds/selecting-data-feeds).

**Вывод:** наше решение Капитана (PAXG/Binance как цена 24/7 + Chainlink сверка) —
лагерь заменителей, совпадает с трендом 2026 (Pyth Indices). Единственная легитимная
альтернатива — остановка торговли на выходные (SCCP-83-модель), она же fallback при
недоступности PAXG. Торговать по замороженной пятничной цене — задокументированно
проигрышная конфигурация. **Конструкция P1-E подтверждена, менять не требуется.**

## Q4 — требования, забираем в P4-A (дословно в спеку P4-A)

1. Один кошелёк = строго последовательная отправка (инвариант keeper:all, Defender-модель).
2. Replace-by-fee: tx не подтверждена за N блоков → ресабмит с тем же nonce и газом выше;
   при хроническом зависании — no-op для проталкивания nonce.
3. Порог долива постера: алерт при балансе < порога (эталон 0.1 ETH; расчётный —
   стоимость прогона при пике газа × 3–5). Алерт ≠ автодолив — долив ручной, по runbook.
4. Метрика «timestamp последнего успешного поста» на фид + алерт
   `now − lastSuccess > HEARTBEAT_AGE + slack` (при прогоне 10 мин — дедлайн ~30 мин).
   Из коробки ни у кого нет — строим сами.
5. Машина состояний прогона (pending→sent→confirmed/reverted/skipped) с persisted
   last-state — поверх наших кодов 0/1/2.
6. Анти-спам: алерт повторяется только при смене состояния.

## Q6 — уточнение к К-1 из разведки

- Роль источника называть «Data Source» (TUSD-прецедент) или «attestor» (Ethena);
  «кастодиан → агрегатор → фид» без формального аудита — принятая практика, но
  Chainlink это честно дисклеймит («non-attest service») — нам тоже стоит.
- PAXG сам PoR-фида Chainlink НЕ имеет: Paxos публикует месячные PDF-аттестации.
  Наш ежесуточный цикл честнее рынка — аргумент для заявки.
- Стандарт при протухании PoR: потребитель проверяет `updatedAt` (Salty-паттерн) +
  Secure-Mint-проверка покрытия при минте (TUSD/Aave). Наш PoR-гард минта — ровно
  второй паттерн, подтверждён.
