# P1-D — Gasless-смоук (CLI): живая спонсируемая транзакция end-to-end

Дата: 2026-07-24 · Инженер · Гейт-1 (план) → после «go»: код → Гейт-2 (диф)
Предусловия выполнены: фиды живы, OLTIN у клиентов есть (P1-C), паймастер
`0x817ED8bd…0Bb4` (0.01 ETH, курс до 2026-08-21, allowlist новых UZD/Exchange
проведён в V3.1). Клиентский контракт — по операционным заметкам
`docs/DEPLOYMENTS.md` (закреплён тестами 4a′).

## Цель

Доказать на проде (Sepolia) то, что до сих пор доказано только zkSync VM-сьюитой:
клиент отправляет транзакцию **без ETH на газ** — газ платит паймастер, клиент
платит комиссию в OLTIN. Это закрывает приёмку Фазы 1 и отложенный пункт PR-4a′.

## Скоуп PR

**Входит:**

1. **`contracts/scripts/smoke-gasless.ts`** — CLI-смоук, один файл, печатает
   каждый шаг и вердикт. Сценарий: клиент-1 делает gasless `transfer` 1 г OLTIN
   клиенту-2 (OLTIN — спонсируемая цель).
   Порядок строго по клиентскому контракту:
   1. `feeData` с провайдера → `maxFeePerGas`;
   2. явный `gasLimit` = estimateGas голого вызова + AA-надбавка (по замеру
      `test-vm/paymaster.e2e.ts`, +150k) — автооценка запрещена
      (`zks_estimateFee` пробует максимальным лимитом и упирается в потолок);
   3. `minimalAllowance = quoteFee(gasLimit, maxFeePerGas) × margin`
      (дефолт 1.2 — approve с запасом, не в точку);
   4. **preflight `checkSponsorship(from, to, gasLimit, maxFeePerGas)`** —
      отказ приходит типированной ошибкой, а не тишиной;
   5. `approve(paymaster, minimalAllowance)` — обычная транзакция клиента
      (у клиента 0.001 ETH на это);
   6. отправка спонсируемой tx с `paymasterParams` и явным `gasLimit`;
   7. **таймаут на майнинг** (дефолт 120 с): отказ — не реверт, а вечное
      ожидание; по таймауту — понятная ошибка «not mined (silently refused)»;
   8. верификация чтением: балансы OLTIN клиентов (1 г переехал), комиссия
      паймастера снята в OLTIN, **ETH-баланс клиента не уменьшился на газ**
      (только approve-газ).
2. **Негативный путь без трат** (eth_call): `checkSponsorship` на
   несуществующую цель → `TargetNotSponsored`; отказ показан типированно.
3. **Юнит-тесты чистых частей** (red→green): расчёт margin-аллоуэнса,
   классификация ошибки таймаута vs реверта. VM-сюита паймастера (4a′) не
   дублируется.
4. **npm-скрипт** `smoke:gasless` + шапка с клиентским контрактом дословно.

**Явно НЕ входит:** UI/webapp (P3-C); смена курса/потолков паймастера;
повторное финансирование паймастера; стейкинг.

## Критерии приёмки

1. **Живой прогон зелёный**: gasless transfer 1 г замайнен; в логе — preflight
   fee, tx hash, время майнинга; ETH клиента убыл только на approve.
2. Комиссия снята в OLTIN (баланс паймастера в OLTIN вырос — чтением).
3. Негативный preflight показывает `TargetNotSponsored` типированно.
4. Юнит-тесты margin/таймаута — красные до, зелёные после; вся сюита зелёная,
   CI зелёный.
5. Приёмка Фазы 1 после этого PR: все пункты зелёные (дашборд — в P2-C по
   решению 24.07).

## Риски

- У клиента-1 ~0.001 ETH: approve + мелочь хватает; спонсируемая tx газа с
  клиента не тратит по построению.
- Курс паймастера жив до 21.08 — в окне конкурса не истекает.
- Если preflight внезапно ревертит caps (per-sender 0.002 ETH/день): одна tx
  `setCaps` владельцем, не передеплой (записано в DEPLOYMENTS).
