# P1-E — Круглосуточная цена золота (медиана PAXG+XAUT) + ослабление гарда UZS

Дата: 2026-07-24 · Инженер · Гейт-1 (план) → после «go» Капитана: код → Гейт-2 (диф)
Следует за P1-A (PR #15). Отдельный PR.

## Цель

Exchange торгует 24/7, а Chainlink XAU/USD — пятидневный фид: на выходных наша
цена заморожена, и мы систематически проигрывающая сторона (разведка
`.claude/research/2026-07-24-keepers-oracles.md`: замороженной пятничной ценой
не торгует никто; ранний Synthetix — задокументированный вектор атаки SCCP-83).
Решение Капитана: **путь заменителя** — живая круглосуточная цена из
токенизированного золота, Chainlink остаётся детектором живости.

## Решения Капитана (нарешано, не открытые вопросы)

1. **Цена = медиана нескольких токенов**: PAXG + XAUT, пары **к USD, не к USDT**
   (USDT отвязан ~0.1% даже сейчас). Источник — **Alchemy Prices API**
   (by-symbol; проверено Капитаном — PAXG/XAUT отдаёт).
2. **Chainlink — только детектор живости, НЕ ценовая сверка.** Расхождение цен
   принято осознанно (на фоне 14% спреда ЦБ — шум): никаких порогов расхождения,
   никакой блокировки по цене. Логика детектора:
   - токен-источник молчит/мусор, а Chainlink свеж → сломались МЫ → тревога
     (refuse, код 1);
   - молчат оба → инфра-авария → тревога (refuse, код 1).
3. **Один провайдер осознанно:** и Prices API, и Chainlink-детектор — через один
   Alchemy-аккаунт (`MAINNET_RPC_URL` + Prices API того же ключа). Резервный RPC
   не заводим; падение провайдера видно по коду 1 кипера и вставшему Exchange,
   чиним по факту.
4. **Публичный экран (требование в P2-A, кода здесь нет):** обе цифры рядом —
   наша медиана и Chainlink — для проверяемости в рассказе жюри.
5. **UZS — ослабить гард источника:** refuse → **warn с ретрансляцией** последнего
   курса; порог «источник сломан» 7 → **14 суток**; heartbeat UZS (сутки) не
   трогаем. Философия та же, что market-closed для XAU: ЦБ держит последний курс
   на выходных/праздниках — это штат, а не авария.
6. **Окна контракта `maxAge*` НЕ трогаем** (передеплой V3, пост-конкурс).
   Частоты киперов — следствие окон, тоже не трогаем.
7. **Линия отреза (в кармане):** если P1-E не влезает — fallback «остановка на
   выходные» (market-hours в кипере XAU, одна env-переменная), обратимо конфигом.

## Скоуп PR

**Входит:**

1. **Источник XAU — медиана токенов** (`keeper-xau.ts` + `keeper-lib.ts`):
   - клиент Alchemy Prices API (by-symbol, PAXG + XAUT, USD), таймаут 30 с;
   - валидация каждого значения: положительное, конечное, разумный диапазон
     (100–100 000 USD — анти-мусор, анти-ноль);
   - медиана валидных: оба → среднее двух (медиана пары); один → он
     (в лог `WARN: degraded to single source <name>`); ноль → детектор (п. 2);
   - результат × 1e8 (dec 8), пост в тот же `XauUsdFeed` — ончейн-контракт
     не меняется.
2. **Chainlink-детектор живости** (тот же прогон keeper-xau):
   - «свеж» = возраст записи ≤ `SOURCE_WARN_AGE` (23 ч — heartbeat 22ч20м + запас);
   - токен-источники недоступны + Chainlink свеж → refuse `token sources dead,
     chainlink alive — WE are broken`; недоступны + Chainlink стар → refuse
     `both sources dead — infra outage`; оба случая — код 1, алерт P4-A;
   - при живых токен-источниках Chainlink не влияет на решение вообще
     (ни сверки, ни порогов); возраст Chainlink логируется информативно.
3. **UZS-гард** (`keeper-uzs.ts`): возраст курса > `MAX_CB_AGE_DAYS` (дефолт
   7 → **14**) → refuse «API сломан»; возраст > `CB_WARN_AGE_DAYS` (дефолт 3)
   и ≤ 14 → **warn + ретрансляция** последнего курса (heartbeat и гард скачка
   работают как раньше). Будни без изменений.
4. **Env** (`.env.example`): `ALCHEMY_PRICES_URL` (с тем же ключом; дефолт —
   шаблон `https://api.g.alchemy.com/prices/v1/$KEY/tokens/by-symbol`),
   `PRICE_SYMBOLS=PAXG,XAUT`, `CB_WARN_AGE_DAYS=3`, `MAX_CB_AGE_DAYS=14`.
   `MAX_SOURCE_AGE`/`SOURCE_WARN_AGE` сохраняются для детектора.
5. **Тесты** (red→green, правило №0) + шапки скриптов + доки.
6. **Ручной прогон Sepolia:** keeper-xau публикует медиану; расхождение с
   текущим ончейн (от Chainlink) ожидается ~0.1%, гард 10% не сработает.

**Явно НЕ входит:** контракты и `maxAge*`; UI «обе цифры рядом» (P2-A);
резервный RPC; пороги расхождения (отвергнуты решением Капитана); режим
market-hours (это fallback, отдельно и только если понадобится).

## Риски

- Alchemy Prices API: форму ответа и поля свежести проверяю живым вызовом
  перед кодом (не из памяти); если `updatedAt` значения нет — свежесть
  токен-источника определяем доступностью ответа, не возрастом.
- Медиана пары = среднее двух: при расхождении источников между собой цена —
  середина; блокировки по этому расхождению нет (решение Капитана), в лог —
  информативно.
- SPOF на Alchemy принят (п. 3 решений).
